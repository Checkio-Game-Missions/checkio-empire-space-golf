Our RoboTrio wants to play golf, but they are too precious and can shot any hole without problem. Thus they changed rules. Now you need pass through all hole in any order.

There are 5 (five) holes on the square field with size 10x10 units. You are given coordinates of the holes. For the winning you should find the shortest way through all holes and your function must return the length of this way. You are start in the point (0, 0) and can finish in any hole. The result will be checked with ±0.01 precision.

In this mission the main goal to make your code as short as possible. The shorter code -- more points. The points for this mission that you get are dynamic and related with the length of your code. We count all characters. 350 characters is the maximum and it's zero points. For each character minus from basis you will get 1 point. For example for 300 characters in code you will get 50 points.

Input: A set of hole coordinates. Each hole coordinates are represented as a tuple of integers.

Output: The length of the shortest path as a float.

Example: 

golf({(2, 2), (2, 8), (8, 8), (8, 2), (5, 5)}) == 23.31
golf({(2, 2), (4, 4), (6, 6), (8, 8), (9, 9)}) == 12.73

How it is used: This task is not about elegant or performance code. This is about hacks and tricks in python which help you to short your code. Of course you don't need to use this in production, but it can help for deep comprehension of python.

Precondition: len(holes) == 5
all(0 < x < 10 and 0 < y < 10 for x, y in holes)
